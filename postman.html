Postman 工具介紹

Postman 是一個可以讓我們測試 HTTP 各個請求的工具。藉由 API 不同請求方式測試請求資料結果是否如我們預期。HTTP 測試結果的回應方式會藉由 HTTP 的代碼反應測試結果。若要知道更多詳細的 HTTP 返回狀態碼定義，請參考 HTTP 返回狀態碼。

Postman 安裝

到 Postman 網站下載並安裝。Postman 安裝有兩個版本，一個是 app 另一個則是網頁版本。如果使用 postman 測試 API 是每天都需要做的事情，可以註冊一個帳號方便紀錄使用過後並且可再次執行的 API，方便之後可再次使用。

  • 1. API 資料請求方法

    • API 資料請求有四大類別分別是新增 (POST) / 取得 (GET) / 更新 (PUT) / 刪除 (DELETE)。若要知道更多詳細的 HTTP 發送方法定義，請參考 HTTP 發送方法。

  • 2. API 資料處理的網址： 這裡輸入所要處理的 API 資源網址。
    例如：https://jsonplaceholder.typicode.com/todos/1。
  • 3. 請求網址參數細節設定： 這裡包含了網址參數設定，驗證碼設定，Header 資料設定和 Body 資料設定。
    • Params (參數): postman 參數設定有兩種，一個是 query params (搜尋參數), 一個是 path variable (搜尋變數)。Postman 開啟時，都只有 query params 可以設定。要讓搜尋變數現身只有在網址輸入 "：{變數名字}" 才可以開啟搜尋變數設定。
      例如： https://jsonplaceholder.typicode.com/todos/1/:new_member
    • Authorization (身份驗證): 如果請求的 API 權限是需要身份驗證才可以請求資料的話，這邊就需要輸入身份驗證的資料。
  • 4. 回傳請求資料結果顯示區塊以及回傳HTTP 返回狀態碼顯示。

什麼是 API

API 是應用程式介面。開發者可以透過 API 把不同的服務串接在一起。例如：我所開發的軟體想要與臉書的服務結合，我可以串接臉書的 API 與我的軟體做一個整合。藉由這樣的互利互惠的操作，可以大幅減少重複開發同樣服務的時間進而增加開發者的生產力。
